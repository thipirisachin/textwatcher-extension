Setup Guide

Get TextWatcher monitoring pages and firing alerts in five steps.

TextWatcher needs OS-level notification permission - browser permission alone is not always enough.

Windows 10 / 11

Settings → System → Notifications → Google Chrome → switch ON

macOS

System Settings → Notifications → Google Chrome → Allow Notifications

TextWatcher only scans pages you tell it about. Without a URL rule, nothing gets monitored.

  1. Open the popup on the page you want to watch
  2. Scroll to Quick Add URL
  3. Click Use Current Tab’s URL — it fills everything in for you
  4. Click Add URL

Match type: Use Exact for a single page, Wildcard (*) for a whole site, or Domain for the entire domain.

Open the popup on your target page, then:

  1. Table Mode — type partial text from columns that identify the row (e.g. a branch name). Watch the preview - aim for 1 unique match
  2. Text Mode — type any keyword or phrase anywhere on the page
  3. Enter a notification title you’ll recognise when the alert fires
  4. Click Save Rule
  1. Click the TextWatcher icon in the toolbar
  2. Flip the toggle in the top-right to ON
  3. Navigate to a monitored page — the dot turns green and shows your active rule count

You can pause at any time - your rules stay intact. TextWatcher scans every open tab that matches a URL rule.

Privacy

By default, all data stays on your device. If you enable a webhook, alert data is sent to your configured endpoint.

Data & Storage

All data stored locally

Keywords, URL rules, settings, and alert history are stored in your browser's local extension storage only.

No network requests by default

The extension makes zero outbound network requests unless you configure a webhook. No analytics, no telemetry, no third parties — ever.

Webhook: alert data leaves your device

When a webhook is enabled, each alert sends a payload to your configured endpoint containing the keyword, page URL, event type, and timestamp. In Generic JSON mode a short text snippet near the keyword match is also included. Teams, Slack, and Telegram formats send metadata only. Delivery uses HTTPS — or HTTP to localhost only for local testing.

No account required

No sign-up, no login, no cloud sync. Your configuration belongs only to you.

Page content never transmitted

Text matching runs entirely in the content script inside your browser. Full page content is never stored or sent. If you use the Generic JSON webhook format, a short text snippet surrounding the match is included in the payload — all other formats (Teams, Slack, Telegram) send metadata only.

Permissions Used

storage

Read and write your keywords, URLs, settings, and alert log to local extension storage.

notifications

Show OS-level alerts when a keyword appears or disappears.

scripting

Inject the content script into matching pages to perform text monitoring.

tabs

Read the active tab URL (for the “Use Current Tab” button) and focus the source tab when you click a notification.

host_permissions: <all_urls>

Allows the content script to be injected into any tab you’re monitoring, and allows the service worker to POST to any webhook URL you configure. The extension only acts on tabs matching your URL rules.
